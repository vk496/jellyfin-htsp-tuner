Tvheadend (HTSP)

These settings drive the integrated Live TV service. They are also the fallback connection for "HTSP Tuner" devices until Jellyfin's UI can show per-tuner fields, so set them here even if you add HTSP under Add Tuner Device.

Tvheadend streaming profile passed to subscribe. Default htsp.

Per-channel in-memory buffer cap. When a client falls behind, oldest data is dropped rather than blocking.
Base URL Jellyfin uses to read the plugin's own live stream. Leave empty to use http://127.0.0.1:<port> (correct for a normal single-host install and avoids Jellyfin picking a wrong network interface). Only set this if Jellyfin's transcoder runs on a different host than its API.

Guide

Copies Tvheadend's channel groupings (Sports, HD, Kids…) onto the Jellyfin channels, where they can be queried as tags — Jellyfin has no genre field for channels. Applied on the next guide refresh.
Most programmes come with no artwork, which is why the Live TV home page is mostly blank placeholder tiles. Every minute, a few of the programmes on air right now that have no picture get a still frame grabbed from the channel itself. A channel someone is already watching is sampled from the stream already open, costing Tvheadend nothing; anything else is tuned at the lowest subscription weight, so a viewer always wins a contended tuner.
Default 200, which is exactly the pool the Live TV home page picks its "On Now" row from — so captures go to what you can actually see. Raise it to reach further down the guide. 0 means every programme on air, which on a Tvheadend carrying thousands of channels means it will spend all day tuning them.
Advanced
Tvheadend pushes EPG changes as they happen. Jellyfin cannot apply just the change — it only offers a full guide rebuild, which reads, diffs and rewrites every channel's programmes and takes many minutes on a large channel list. So this is a rate limit, not a poll: a push only triggers a refresh once the guide is at least this old. Default 720 (12 hours). 0 disables it and leaves the guide to Jellyfin's own schedule. Setting it low will queue refreshes back-to-back and keep your server busy for nothing.
How long ffmpeg inspects a channel before playing it. Lower = a channel starts faster; too low risks a bad start on an unusual stream. Default 2000. Most of the start delay is Jellyfin's own HLS buffering, which this does not affect — so expect at most about a second of improvement. Range 250–10000.
